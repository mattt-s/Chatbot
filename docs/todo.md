# Todo / 迭代计划

## 一、群组功能补全

### 角色工具权限分离
当前所有角色都是 `tools.profile: "full"`，按角色限制工具集（rd 只有代码工具、ui 只有设计工具等），减少意外操作，也是 Harness engineering 的实践。

### 空回复 / 确认回复过滤
提示词里禁止了纯确认回复，但没有机制检测违规。在 ingest 层识别"收到/好的/明白"类回复，静默过滤或标记，避免噪声消息进入对话流。

### 角色静默检测
Watchdog 目前只催 leader 进度，但某个成员卡死（既没回复也没更新记忆）也无法感知。检测成员 N 轮无响应，自动通知 leader。

---

## 二、可观测性 / 调试体验

### 群组对话时序视图
目前调试需要 SSH 到服务器查 JSONL 日志，体验很差。在 UI 里做"群组时序视图"，展示每条消息的发送角色、时间、runId，帮助理解群组协作过程。

### ~~记忆板 UI~~（已完成）
群组面板菜单新增"查看记忆板"入口，弹窗展示各角色当前记忆内容和更新时间。`groupMemory` 已纳入 `PanelView`，每次角色回复完成后随 `refreshPanelMeta` 自动刷新。

---

## 三、可靠性 / 长任务支持

### 错误自动恢复
某个角色 `state=error` 时目前只是停住，没有后续处理。让 leader 收到错误通知，决定是否重试或调整方向。

### 群组任务持久化
长任务跨会话继续（用户关掉浏览器后第二天回来继续）目前没有明确支持。需要把任务上下文和进度写入持久存储，重新打开能续上。

---

## 四、能力扩展

### 用户上传文件给群组
目前用户只能发文字，角色间通过文件路径共享文件。支持用户直接上传文件（图片、文档、代码）交给群组处理，扩大应用场景。

### 定时触发群组任务
结合 openclaw 的调度能力，定时启动群组任务（每天早上生成日报、定期代码审查等）。

### 单角色对话升级为群组
在 direct 对话中发现任务复杂时，一键把当前对话上下文带入新建的群组继续处理。
